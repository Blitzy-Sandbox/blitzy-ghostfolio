Feature A

Snowflake Sync Layer

An append-only analytical mirror of every portfolio change.

Feature A — Snowflake Sync Layer

What It Does

  • Daily cron at 02:00 UTC
  • Event-driven via PortfolioChangedEvent
  • MERGE upserts (Rule 7 — idempotent)
  • Bind variables only (Rule 2 — no SQL injection)

Three Snowflake Tables

  • portfolio_snapshots
  • orders_history
  • performance_metrics

The sync layer’s queryHistory(userId, sql, binds) method is also consumed by the AI Chat Agent’s query_history tool — one bind-variable code path serves both batch sync and ad-hoc analytics.

02

Feature B

AI Portfolio Chat Agent

Streaming Claude that knows your portfolio.

Feature B — AI Portfolio Chat Agent

API Surface

  • POST /api/v1/ai/chat
  • Server-Sent Events streaming
  • JWT-authenticated (AuthGuard('jwt'))
  • Stateless server-side; client carries 4 prior turns

Four Tool Calls

  • get_current_positions
  • get_performance_metrics
  • query_history → Snowflake
  • get_market_data

The system prompt is personalized per request with the caller’s portfolio state and FinancialProfile; tool-supplied userId is always overridden by the JWT-verified user (Rule 5).

Feature C

Explainable Rebalancing Engine

Every recommendation cites the goal it serves.

Feature C — Explainable Rebalancing Engine

How It Works

  • POST /api/v1/ai/rebalancing
  • Single JSON payload (no streaming)
  • tool_use exclusively (Rule 4)
  • No text-content parsing

Per-Recommendation Schema

  • action: BUY / SELL / HOLD
  • ticker, fromPct → toPct
  • rationale (plain language)
  • goalReference (FinancialProfile field or goal label)

A single Anthropic tools array element with tool_choice: { type: 'tool', name: 'rebalancing_recommendations' } forces Claude to emit a structured object — never free-form text.

Risks

And How We Mitigate Them

Anticipated failure modes and the controls that contain them.

Risk & Mitigation Register

Risk Likelihood Mitigation
Anthropic API outage Low /api/v1/health/anthropic probe + degraded UX with reconnect button (Rule 6).
Snowflake credential leakage Low ConfigService only (Rule 3); structured logger redacts SNOWFLAKE_PASSWORD and bind values that resemble credentials.
SQL injection via tool input Low Bind variables exclusively (Rule 2); semicolon rejection in query_history; row-count cap on every SELECT.
LLM hallucination of trade actions Medium Forced tool_use JSON shape (Rule 4) + structural validation; warnings array surfaced in every response.
Cross-user data leak Low JWT-derived userId overrides any tool-supplied value; where: { userId } on every Prisma query (Rule 5).
Duplicate Snowflake rows on re-sync Low MERGE upserts keyed on documented unique constraints (Rule 7); cron + event handler converge on the same code path.
SSE stream disconnect Medium Non-empty errorMessage + visible reconnect button in ChatPanelComponent (Rule 6).

Onboarding

Where to Start

A 5-minute orientation for new engineers.

Get Started in Five Steps

  1. Read the Agent Action Plan. The full mission briefing — scope, rules, gates — lives in the PR description.
  2. Populate seven new env vars. See .env.example for the six SNOWFLAKE_* entries and ANTHROPIC_API_KEY.
  3. Run npm install + npx prisma migrate dev. Picks up the three new packages and creates the FinancialProfile table.
  4. Start the stack with docker compose up + npm run start:dev. Exercises the SSE chat at /portfolio and the rebalancing page at /portfolio/rebalancing.
  5. Review the eight rules and 13 gates. See AAP § 0.7 for the non-negotiable engineering contract.

Questions? Drop them in the #ai-portfolio Slack channel.
